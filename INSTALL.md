# Install

This project is deployed from the repo root.

## Requirements

- Python 3.11+
- `venv`
- Telegram bot token
- Your Telegram numeric user id added to `ADMIN_IDS`
- One or more VPS/server nodes reachable over SSH

Optional, depending on deployment mode:

- Docker and Docker Compose, if you want to run the bot in containers
- systemd, if you want to run the bot as a regular host service

## 1. Prepare the bot host

```bash
git clone <your-repo-url> vpn-manager-bot
cd vpn-manager-bot
cp .env.example .env
mkdir -p data ssh
```

Edit `.env` and set at least:

```env
BOT_TOKEN=...
ADMIN_IDS=123456789
SSH_KEY=/root/.ssh/id_ed25519
```

## 2. Choose a deployment mode

You can run the bot either:

- in Docker with `docker compose`
- as a regular process managed by `systemd`

### Option A: Docker

```bash
docker compose up -d --build
```

### Option B: systemd

Create a virtualenv and install dependencies:

```bash
python3 -m venv .venv
. .venv/bin/activate
pip install -r requirements.txt
mkdir -p data ssh
```

Initialize the SQLite schema once:

```bash
.venv/bin/python app/manage_db.py init
```

Create `/etc/systemd/system/vpn-manager-bot.service`:

```ini
[Unit]
Description=vpn-manager-bot
After=network-online.target
Wants=network-online.target

[Service]
Type=simple
WorkingDirectory=/opt/vpn-manager-bot
Environment=VPN_BOT_BASE_DIR=/opt/vpn-manager-bot
EnvironmentFile=/opt/vpn-manager-bot/.env
ExecStart=/opt/vpn-manager-bot/.venv/bin/python /opt/vpn-manager-bot/app/main.py
Restart=always
RestartSec=3

[Install]
WantedBy=multi-user.target
```

Then enable and start it:

```bash
sudo systemctl daemon-reload
sudo systemctl enable --now vpn-manager-bot
sudo systemctl status vpn-manager-bot
```

## 3. Prepare SSH access for node management

Inside Telegram:

1. Open the bot as admin
2. Open `Admin -> SSH Key`
3. Copy the generated public key
4. Add it to `~/.ssh/authorized_keys` on the target node

## 4. Register and bootstrap the first node

Inside Telegram:

1. Open `Admin -> Servers`
2. Add a server
3. Open the server card
4. Run `Probe`
5. Run `Bootstrap`

What bootstrap currently does:

- installs base packages
- ensures Docker is available on the node
- prepares node config files
- deploys Xray as a container
- deploys AWG as a container
- installs helper scripts and node runtime files under `/opt/vpn-manager-node`

## 5. Create the first profile

Inside Telegram:

1. Open `Admin -> Profiles`
2. Create a profile
3. Select one or more protocols/server methods
4. Set subscription duration

## 6. Test from a normal account

From another Telegram account:

1. Open the bot
2. Use `Get Key`
3. Test:
   - Xray URI + QR
   - AWG `vpn://` + QR
   - AWG `.conf` fallback

## Notes

- If the bot itself runs inside Docker, host nodes should be added via `ssh`, not `local`
- SQLite lives in `data/bot.sqlite3`
- SSH keys generated by the bot live in `ssh/`
- In `systemd` mode, the bot still manages VPN nodes separately under `/opt/vpn-manager-node`
- For public deployment, review `.env`, `.gitignore`, and tracked files before pushing
